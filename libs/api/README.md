# Модуль Сетевого Взаимодействия Ядра (@cabinet/api)

## Назначение
Изолированная инфраструктурная библиотека монорепозитория **Cabinet**. Инкапсулирует логику HTTP-запросов и предоставляет интерфейсы для ЛК Operator и ЛК Developer.

## Токены Инверсии Зависимостей (DI Tokens)
Для реализации гибкой подмены и композиции на этапе компиляции используются два токена:
1. `API_IMPLEMENTATION` — Глобальный токен рантайм-реализации для всего приложения.
2. `PARTNER_API_IMPLEMENTATION` — Токен для базового API White Label партнера. Позволяет `DevApiService` динамически подключаться к любому партнеру и делегировать ему стандартные запросы (`getPayments`).

## Флоу вызовов в рантайме
Подробный сценарий прохождения запросов от UI-компонентов через Фасад до API-шлюза описан в файле `api-sequence.puml`. Разработчикам запрещено внедрять конкретные классы сервисов в компоненты напрямую.
